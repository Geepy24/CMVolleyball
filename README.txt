1.网站首页是自己做的
2.后台的网站资源管理系统的前端页面是直接copy其他项目更改而来
3.添加了文本编辑器，由jq22.com下载而来
4.文章管理系统(管理员):
	1)管理员文章发布功能
	2)文章的查询功能
	3)文章的删除与恢复功能（回收站）
	4)文章的详细信息功能
	5)修改文章的内容
	6)文章的临时保存功能
5.文章的数据库表结构：
	文章id
	文章作者（usrName）
	作者id（userId）
	管理员（ads_name）
	文章标题
	发表时间
	存储的uri地址
	最后一次修改的时间
6.回收站的数据库表结构
	被删除的文章id
	被删除的文章作者
	被删除的文章标题
	删除的时间
	被删除的文章的uri地址
7.资源的数据库表结构
	资源id
	资源类别
	资源的自定标签
	上传者的id（userId）
	上传者的name（userName）
	管理员（ads_id）
8.用户-文章-资源对照表
	用户id
	文章id
	文章标题
	文章uri
	资源id
	资源类别
	资源uri
问题：插件中编辑器内图片与视频上传到哪里

		没有使用一对多的表结构引入外键，都是使用单表。
		用了用户，文章，资源的对应表来查找某个用户上传的文章和资源，减小查询压力
9.管理员发表文章逻辑
	1）前端用json格式提交文章的标题、时间、内容
	2）动作类模型驱动接收json格式数据，1 写出到服务器的指定目录下新建的jsp页面中，并获得uri地址
			2 封装好实体类article对应的数据，保存到数据库的article表 
			3 返回uri的json格式数据到前端
	3）前端接收返回的数据，1添加该uri对应链接并加上标题、作者、日期显示到“新闻总览”（新页面）中
			2 文章总览的内容按照时间先后顺序显示到首页的“球队新闻”栏目下
